# Sprint 6 — WMS (Rust Render Engine) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** WMS service in Rust: CQL→PostGIS SQL (byte-identical to Go, proven by shared corpus), SLD 1.0/1.1 parse+apply, a tiny-skia render pipeline with cosmic-text labels, GetMap/GetFeatureInfo/GetLegendGraphic/GetCapabilities for WMS 1.1.1+1.3.0, vector data from PostGIS, fast PNG/JPEG/WebP output.

**Architecture:** Rendering-heavy code in Rust (spec §10). `geo-core` gains a `filter` module (CQL parser + PostGIS SQL emitter) that must pass the same `tests/filter-corpus/corpus.json` the Go side passes — this is the drop-in guarantee that CQL behaves identically across services. `services/wms` (Rust, axum, reuses `geo_core::health`) resolves layers from the catalog Postgres, queries features via `sqlx`, maps geometries with `geozero`, renders with `tiny-skia` + `cosmic-text`, and encodes with `image`. SLD drives symbolizer choice.

**Tech Stack:** Rust stable, axum 0.8, tokio, sqlx (postgres, runtime-tokio), geozero (+wkb), tiny-skia 0.11, cosmic-text 0.12, image 0.25 (png fast + jpeg + webp), roxmltree 0.20 (SLD parse), serde_json.

## Global Constraints

- Health convention: reuse `geo_core::health::router`; `GEOSON_HTTP_ADDR` default `:8080`; SIGTERM graceful shutdown (wms stub pattern from Sprint 1).
- Rust: `cargo test --workspace`, `cargo fmt --all --check`, `cargo clippy --workspace -- -D warnings` all pass (CI gates). Integration tests read `GEOSON_TEST_DATABASE_URL`, skip when unset (compose Postgres `127.0.0.1:5433`).
- Filter parity: `geo-core` CQL→SQL must produce byte-identical output to Go `libs/ogc-kit/filter` for every case in `tests/filter-corpus/corpus.json`. The corpus is the contract; do not diverge — fix the impl, not the corpus.
- SQL safety: parameterized values (sqlx bind), identifiers validated `^[A-Za-z_][A-Za-z0-9_]*$` + double-quoted. Same rule as Go side.
- Gateway proxies `/geoserver/.../wms` → `wms:8080/wms` with `X-Geoson-Workspace/Layer/Version/CQL-Read` headers. WMS honors them.
- Vendor params (case-insensitive): `LAYERS`, `STYLES`, `BBOX`, `WIDTH`, `HEIGHT`, `SRS`/`CRS`, `FORMAT`, `TRANSPARENT`, `BGCOLOR`, `CQL_FILTER`, `TILED`, `EXCEPTIONS`. Axis order: WMS 1.3.0 + EPSG geographic → lat/lon (y,x) BBOX; 1.1.1 → lon/lat (x,y).
- Docker: Rust build stage needs GDAL later (Task 8); base tasks stay pure-Rust alpine until then. `image` PNG uses fast mode where available.
- Commit after every task, Conventional Commits.

#[cfg(test)]
mod tests {
    use super::super::{parse_cql, to_sql, ast::Arg};
    use serde_json::Value;

    #[test]
    fn golden_corpus_matches_go() {
        let path = concat!(env!("CARGO_MANIFEST_DIR"), "/../../tests/filter-corpus/corpus.json");
        let raw = std::fs::read_to_string(path).expect("read corpus");
        let cases: Vec<Value> = serde_json::from_str(&raw).unwrap();
        assert!(cases.len() >= 30, "corpus too small: {}", cases.len());
        for c in &cases {
            let cql = c["cql"].as_str().unwrap();
            let want_sql = c["sql"].as_str().unwrap();
            let e = parse_cql(cql).unwrap_or_else(|err| panic!("{cql}: {err}"));
            let (sql, args) = to_sql(&e, 1).unwrap_or_else(|err| panic!("{cql}: {err}"));
            assert_eq!(sql, want_sql, "SQL mismatch for {cql}");
            let want_args = c["args"].as_array().unwrap();
            assert_eq!(args.len(), want_args.len(), "arg count for {cql}");
            for (got, want) in args.iter().zip(want_args) {
                assert!(arg_eq(got, want), "arg mismatch for {cql}: {got:?} vs {want}");
            }
        }
    }

    fn arg_eq(got: &Arg, want: &Value) -> bool {
        match (got, want) {
            (Arg::Str(s), Value::String(w)) => s == w,
            (Arg::Bool(b), Value::Bool(w)) => b == w,
            (Arg::Num(n), Value::Number(w)) => (*n - w.as_f64().unwrap()).abs() < 1e-9,
            _ => false,
        }
    }
}
```

#[derive(Debug, Clone, Default)]
pub struct Style { pub rules: Vec<Rule> }

pub struct Rule {
    pub min_scale: Option<f64>, pub max_scale: Option<f64>,
    pub symbolizers: Vec<Symbolizer>,
}
#[derive(Debug, Clone)]
pub enum Symbolizer {
    Point { fill: [u8;4], size: f32, well_known: String },
    Line { stroke: [u8;4], width: f32 },
    Polygon { fill: [u8;4], stroke: [u8;4], stroke_width: f32 },
    Text { property: String, fill: [u8;4], size: f32 },
    Raster { opacity: f32 },
}
/// parse_sld reads SLD 1.0 (sld ns) or 1.1 (se ns) into a Style.
/// Unknown symbolizers are skipped; empty style -> a default gray polygon rule.
pub fn parse_sld(xml: &str) -> Result<Style, String>;
pub fn default_style_for(geom_type: &str) -> Style; // fallback per geometry
```

CssParameter/SvgParameter `fill`/`stroke`/`stroke-width`/`fill-opacity` → RGBA (`#RRGGBB` + opacity → alpha). PointSymbolizer Mark/WellKnownName/Size; LineSymbolizer Stroke; PolygonSymbolizer Fill+Stroke; TextSymbolizer Label/ogc:PropertyName + Font size + Fill; RasterSymbolizer Opacity. Handle both `sld:` (1.0) and `se:` (1.1) prefixes by matching local names.
